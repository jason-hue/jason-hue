如何入侵蓝牙设备

  • 2020-08-08

首先，我们来看看今天的主角——一个非常有用的攻击媒介（蓝牙）

蓝牙设备的出现让我们的生活更加便利！在2020年，你可以在任何一个人家中发现蓝牙设备，廉价的基础蓝牙设备几乎人人都有、人人都用，新的蓝牙设备也在不断被推上市场。总体来说，有谁会不喜欢快捷便利的蓝牙设备呢？Who doesn’t？但是，如果有一个使用蓝牙音箱外放的讨厌邻居或表面风平浪静、尽职尽责，实际上却偷偷使用蓝牙耳机听音乐的下属简直太糟糕可怕了！今天，让ThousandShadow带你对蓝牙设备进行DDoS攻击，解决这些令人心烦的问题吧。

尽管现代的Wi-Fi路由器能够过滤不需要的数据包，但大多数蓝牙适配器还是比较笨拙的。总的来说，他们并不关心您发送的是什么包，多少包。因此，对于我们来说，将Linux中ping数据包的信息量增加到一个巨大的值，然后将这些数据包发送到具有蓝牙功能的设备（例如588）绝对不困难。

关于现成的干扰机

好奇的商品在互联网上出售，您可以在其中找到频率干扰器。他们通常花费不菲的钱，并提供不同的选择。有些可以立即关闭移动网络，Wi-Fi和蓝牙的几乎所有信号，而另一些则无法应对H +。

如果您认为确实需要类似的产品，那么强烈建议您在购买之前熟悉当前的法规。在中国，除有特殊涉密任务单位之外的单位和个人，不得使用无线电干扰设备。凡符合使用无线电干扰设备要求条件的涉密单位，需购置、设置和使用干扰设备的，需持所在市（地）级以上保密部门或中央和国家机关保密工作机构审核批准的《保密会议移动通信干扰器购置审核备案表》，按批准的设备型号和数量向国家保密局授权的企业购买，接受无线电管理部门的指导、监督和检查，并在使用前必须到无线电管理部门备案后方可使用。

我们连接到其他人的设备

众所周知，原始扬声器和头戴式耳机几乎从不过滤发送给他们的数据包。但是，如果您不仅向此类设备发送带有ping数据的数据包，而且向其发送请求连接到该设备的数据包，该怎么办？并不困难。

并非所有耳机制造商都已注意缓冲区溢出保护。如果所有程序包都已排队，那么无处存储请求时会发生什么？该列将尝试执行命令并刷新缓冲区。

因此，我们使用标准的蓝牙通信协议-rfcomm。但是由于控制协议本身的实用程序不允许我们发送上千个这样的请求，因此我们将编写一个小的Python脚本来自动执行该过程。

#!/usr/bin/env python
import subprocess
cmd=['rfcomm', 'connect', '<MAC_address>', '1']
for i in range(0, 1001):
subprocess.call(cmd)
print('Connecting...')

在运行脚本之前，我们需要找出设备的MAC地址。为此，我们使用已经熟悉的命令hcitool scan，然后将接收到的地址插入到脚本中<MAC_address>。现在，我们保存脚本并运行它：

$ python <FileName>

该脚本是否有效取决于被攻击设备的型号，或者取决于填充缓冲区的速度。如果缓冲区在清除之前有时间填满，则将执行该命令，然后我们将与主用户同时连接到该列。如果缓冲区被清除，则将无法执行任何操作，您将不得不重新启动脚本。

如果我们的脚本成功运行，我们便可以拥有自己的迪斯科舞厅或关闭他人的蓝牙设备。
